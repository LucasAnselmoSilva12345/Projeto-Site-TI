Aplicação C#

C# foi programado em varias versões,para conseguir chegar no objetivo encontrado.

Versão 1.0

Alia o poder de C++ com a simplicidade do Visual Basic, a primeira versão vinha com as seguintes categorias.

  • Código gerenciado com a noção de assemblies carregando o bytecode com segurança embutida e controle de versões;

  • Orientação a objeto;

  • Tudo deriva de Object;

  • Os tipos são Estruturas, Enumerações, Classes, Interfaces, Delegados e Ponteiros;

  • Tipagem estática;

  • Eventos;

  • Interoperabilidade com código não gerenciado através de DLL nativa e componentes COM;

  • Coletor de lixo de memória baseado em gerações;

  • Compilador Just-in-time;

  • Facilidades de integração com ferramentas e IDE;

  • Sintaxe para documentação;

  • Sistema de exceção uniforme;

  • Mecanismo de metadados;

  • Extensiva biblioteca padrão;

Versão 2.0

A versão 2.0 teve a atualização de "tipos", varios métodos de atributos de c#, e mais classes e acessibilidade

  • Tipos e métodos genéricos;

  • Tipos Parciais;

  • Métodos anônimos;

  • Iteradores;

  • Tipos anuláveis;

  • Acessibilidade separada para Getter/setter;

  • Conversões de grupo de métodos para delegados;

  • Co e Contravariância para delegados;

  • Classes estáticas;

  • Inferência de delegado;

Versão 3.0

Iniciação de objetos, e também de variaveis

  • Variáveis locais implicitamente tipadas;

  • Inicializadores de objetos e coleções;

  • Propriedades auto-implementadas;

  • Tipos anônimos;

  • Métodos de extensão;

  • Expressões de consulta;

  • Expressões Lambda;

  • Árvores de expressão;

  • Métodos parciais;

Versão 4.0

Inicio de vinculos e mais opções de temas

  • Vinculação dinâmica;

  • Argumentos nomeados e opcionais;

  • Co e contravariância genérica;

  • Tipos embutidos de interoperabilidade ("NoPIA");

Versão 5.0

Métodos e Atributos

  • Métodos assíncronos;

  • Atributos de informação do chamador;

Versão 6.0

Compiladores, dicionários e Iniciação dos primeiros programas

  • Compilador como serviço (Roslyn)

  • Importação de membros de tipos estáticos no espaço de nomes

  • Filtros de exceção

  • Await em blocos catch/finally

  • Inicializadores de propriedades automáticas

  • Valores padrões para propriedades apenas getter

  • Membros com corpos como expressão

  • Propagador de Null (operador condicional de null, checagem de null sucinta)

  • Interpolação de string

  • Operador nameof

  • Inicializador de dicionário

Versão 7.0

Características em processo de desenvolvimento

  • Literais binários

  • Separadores de dígitos

  • Funções locais

  • Switch para tipos

  • Retorno de referências

  • Tuplas

  • Declaração Out (Out var ou Out "tipo")

  • Pattern Matching

  • Returno de async arbitrário

  • Registros (Record)

Características de C#
Dentre as características essenciais do C# podemos citar: Características
Simplicidade os projetistas de C# costumam dizer que essa linguagem é tão poderosa quanto o C++ e tão simples quanto o Visual Basic;
Completamente orientada a objetos: em C#, qualquer variável tem de fazer parte de uma classe;
Fortemente tipada: isso ajudará a evitar erros por manipulação imprópria de tipos e atribuições incorretas;
Gera código gerenciado assim como o ambiente .NET é gerenciado, assim também é o C#;
Tudo é um objeto: System.Object é a classe base de todo o sistema de tipos de C#;
Controle de versões: cada assembly gerado, seja como EXE ou DLL, tem informação sobre a versão do código, permitindo a coexistência de dois assemblies homônimos, mas de versões diferentes no mesmo ambiente;
Suporte a código legado: o C# pode interagir com código legado de objetos COM e DLLs escritas em uma linguagem não-gerenciada;
Flexibilidade se o desenvolvedor precisar usar ponteiros, o C# permite, mas ao custo de desenvolver código não-gerenciado, chamado “unsafe”;
Linguagem gerenciada: os programas desenvolvidos em C# executam num ambiente gerenciado, o que significa que todo o gerenciamento de memória é feito pelo runtime via o GC (Garbage Collector)
